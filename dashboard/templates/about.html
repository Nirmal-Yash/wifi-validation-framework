Automated Regression Testing Framework

NetRegress is a bespoke, 7-layer automated testing framework designed specifically to validate Wi-Fi Data Link (L2) and Network (L3) behaviors across simulated firmware updates. By executing a comprehensive suite of feature tests within a localized virtual environment, the framework ensures no firmware update silently breaks previously working functionality.

The 7-Layer Architecture

1. Virtual Foundation

Replaces resource-heavy VMs with isolated Linux Network Namespaces (netns) and kernel-level software-defined radios.

2. Configuration Manager

Utilizes dynamic YAML files to standardize test variables, authentication thresholds, and device target mapping.

3. Device Connectivity

Executes high-speed OS-level subprocess hooks and interactive CLI prompt handling to manage device states.

4. Traffic Validation

Leverages asynchronous tcpdump packet captures, parsed natively in Python using Scapy, alongside iperf3 benchmarking.

5. Execution Engine

Powered by Pytest fixtures. Automates the full lifecycle of hostapd daemons, DHCP servers, and client connections.

6. Regression Intelligence

A custom diff engine natively compares historical SQLite test results to actively flag Pass → Fail deltas across updates.

7. Dashboard & Reporting

This Flask-powered web portal. It visualizes data plane analytics, highlights recent failures, and maps the virtual topology directly from the execution database.
